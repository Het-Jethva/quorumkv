# Deduplicate mutations in replicated state

QuorumKV will give retried mutations at-most-once effects within an explicit client session by attaching its leader-generated identity and a monotonically increasing sequence number to each logical mutation and retaining the latest sequence and result in replicated state. Sequence numbers begin at one: the next sequence is applied and cached, a duplicate of the latest sequence returns its cached result, and stale or skipped sequences are rejected. Many sessions may operate concurrently, but each session may have only one mutation in flight, which bounds deduplication state per session. Session open and close operations are replicated; closed session identities cannot be reused, and v1 uses a configured active-session limit rather than automatic clock-based expiry, so crashed clients may leave orphaned sessions. Deduplication therefore survives leader failover, log replay, and snapshot installation, at the cost of additional protocol and state-management complexity; QuorumKV will not claim exactly-once message delivery.
